Endpoint Controls

Blocked Endpoints

Blocked endpoints return an error to everyone in real-time. Use a full path (games.roblox.com/v1/games) to block it and everything under it, or just a host (games.roblox.com) to block the whole service. Use * as a wildcard for one path segment: games.roblox.com/v1/games/*/servers blocks servers for any game but leaves /votes alone.

Pattern Note Added

Per-Endpoint Rate Rules

Limit how often a single IP can hit an endpoint (e.g. 1 every 60s). This only ever makes an endpoint more restrictive — the global per-IP limit still applies on top. Supports * wildcards too, e.g. games.roblox.com/v1/games/*/servers.

Request Filters (Header Blocking)

Deny requests by their headers — useful for blocking exploit tooling that tags its traffic (e.g. a Xeno-Fingerprint header, or a User-Agent containing Xeno). Blocked callers get a generic Not Found with no hint as to why, so they can't reverse-engineer the rule. Matching is case-insensitive.

  • Header name — matches the header's key (e.g. block any request sending an Exploit-Guid header).
  • Header value — matches the header's value (e.g. block a User-Agent value containing Xeno).
  • Name or value — trips if either side matches (one rule catches Xeno-Fingerprint and User-Agent: Xeno/…).
